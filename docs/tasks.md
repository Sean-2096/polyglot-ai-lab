# Polyglot AI Lab - 学习路线

## 1. 执行规则

- 同一时间只有一个主要 AI 任务。
- 先完成实验，再开启下一个概念。
- 英语任务依附当前 AI 实验。
- 日语任务每天独立完成，不阻塞 AI 实验。
- 每个任务以“能够重建和解释”为完成标准。
- 日常教学采用英语优先的对话式引导。
- AI 每次只给一个问题或小任务，用户先回答。
- 每个完整学习单元结束后保存简短课程总结。
- 每节课结尾加入 1～3 个零基础日语学习项。

## 2. Stage 0：建立基线

### L0-01 学习时间与能力基线（已完成）

- 记录一周实际可用时间。
- 完成一个短 Python 任务，确认语言基础。
- 阅读一小段英文技术资料，记录理解方式。
- 日语：认识平假名、片假名、汉字三种文字。

### L0-02 建立实验模板

- 创建第一个实验目录。
- 建立 example、modify、rebuild、test、reflection 模板。
- 验证可以在本机运行 Python 和 pytest。

### L0-03 建立帮助协议

- 约定 AI 助手的五级提示。
- 用一个小错误练习“观察 → 提示 → 修复 → 复述”。

**完成门槛**：能够独立运行模板、测试和复盘流程。

## 3. Stage 1：第一次模型调用

### L1-01 观察完整示例

- 使用官方 SDK 发送最小请求。
- 打印模型、消息和响应。
- 画出数据流。

### L1-02 单变量实验

- 分别修改 message、temperature 或 model。
- 每次运行前先记录预测。
- 比较实际输出。

### L1-03 从空文件重建

- 不查看 example。
- 实现模型调用和最小错误处理。
- 通过 Fake 客户端测试。

### L1-04 英语复述

- 阅读一份官方 quickstart。
- 回答三个具体问题。
- 写 3～5 句英文总结。

**完成门槛**：能解释请求各字段，并从空文件完成调用。

## 4. Stage 2：可靠性

### L2-01 超时和错误

- 模拟认证失败、超时和限流。
- 区分可重试与不可重试错误。

### L2-02 结构化输出

- 定义一个小 Schema。
- 验证正确、缺字段和类型错误输出。

### L2-03 Prompt 对比

- 固定输入，只改变 Prompt。
- 使用小评估集比较结果，而不是凭感觉判断。

### L2-04 重建与迁移

- 从空文件实现结构化调用。
- 将 Schema 换成另一个真实问题。

**完成门槛**：能构建一个经过校验、有失败处理的模型调用。

## 5. Stage 3：检索

### L3-01 手算相似度

- 用极小向量理解距离与排序。

### L3-02 Embedding 实验

- 比较相关和不相关文本的相似度。

### L3-03 Chunk 与 Retrieval

- 对同一材料尝试不同切分方式。
- 观察漏召回和错误召回。

### L3-04 带来源回答

- 只根据检索片段回答。
- 缺少依据时拒绝猜测。

**完成门槛**：能解释 RAG 每一步，并定位一次检索失败。

## 6. Stage 4：工具与 Agent

### L4-01 一个确定性工具

- 定义参数、返回值和错误。

### L4-02 最小 Tool Loop

- 实现模型选择工具、执行工具和返回结果。

### L4-03 状态与停止条件

- 防止无限循环和重复调用。

### L4-04 评估

- 建立成功、失败和不应调用工具的案例。

**完成门槛**：能够解释哪些决策交给模型，哪些必须由程序控制。

## 7. 日语零基础路线

日语与上述 Stage 并行，但不绑定具体 AI 任务。

### J0：声音与文字系统

- 日语三种文字。
- 元音和基本发音。

### J1：平假名

- 每次 3～5 个。
- 识别、听音、输入和间隔复习。

### J2：片假名

- 优先学习常见外来语。

### J3：固定表达

- 问候、自我介绍、感谢、确认。
- 使用假名和中文辅助。

### J4：基础句型

- は、の、を、に等基础助词。
- 极短可理解输入。

**项目接入门槛**：平假名识别稳定，且能在不依赖罗马音时理解少量固定表达。
